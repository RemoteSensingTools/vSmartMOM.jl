## What didn't change: the SWIR2 plot

The refactor was structurally clean but had **near-zero numerical effect** on output spectra. SWIR2 percentile ratios MODTRAN/vSmartMOM (5/25/50/75/95): **3.2× / 4.4× / 6.1× / 7.3× / 4.9×** — identical to the pre-refactor run.

**Why:** in the old code, forward H2O line absorption already used `profile.vmr["H2O"]` from YAML × `profile.vcd_dry`, which gave the right column even with q=0 (because `vcd_dry` with q=0 was the full air column, and the YAML `vmr.H2O` was correct). MT-CKD also used `profile.vmr["H2O"]`. The only delta after the refactor is the ~0.5% surface correction to `vcd_dry` from the now-nonzero `vmr_h2o`. Invisible on the plot.

So: the SWIR2 gap is real, but it's **not** caused by H2O absorption being absent. The user's `q=0 → H2O missing` hypothesis was wrong. Refactor still worth doing for code clarity.

## Pickup for tomorrow

**Investigate H2OSTR scaling in [test/benchmarks/emit_modtran_noRS_scenarios.jl](file:///home/sanghavi/code/github/uni_vSmartMOM/test/benchmarks/emit_modtran_noRS_scenarios.jl).** The script comment ("Inner loop (H2OSTR × AOT550) still uses direct τ mutation for speed") suggests it scales `τ_abs` directly per scenario. The dry-scenario H2OSTR=0.05 → ~6% scale of YAML nominal column. If the scale is being applied to the full τ_abs (including non-H2O species and continuum) instead of only the H2O contribution, we'd see exactly the kind of uniform ratio gap we observe (3–7× across all percentiles). **Check first.**

Other suspects, lower priority:
- **Spectral grid undersampling** at 0.1 cm⁻¹: SWIR2 line FWHMs are 0.005–0.05 cm⁻¹; line cores get clipped. Would need a finer band grid for SWIR2 specifically.
- **HITRAN edition mismatch.** What edition was the LUT built from? MODTRAN typically uses HITRAN 2020.

Numerical baseline at single diagnostic point (λ=2184.93 nm, H2O=0.05, AOT=0.0001, GNDALT=0, TSZ=168):
- Full-column model τ_abs (no H2OSTR scaling, fresh model build): **1.69**
- Plotted vSmartMOM τ_atm (after H2OSTR scaling): **0.072**
- MODTRAN τ_atm same scenario: **0.208**

So the H2OSTR scaling takes 1.69 → 0.072 (factor 0.043) — that's *more* aggressive than the naive 0.059 ratio (0.05 / 0.85 g/cm²). This is the smoking gun to chase first.

## What changed today

1. **Documented the azimuth convention** in every YAML
   (`test/test_parameters/*.yaml`, `config/*.yaml`,
   `src/CoreRT/DefaultParameters.yaml` — 41 files). Comment block goes
   directly above the `geometry:` line. After an initial wording error
   the wording was corrected to:
   - `vaz = 0°`  → Sun and satellite on **opposite** sides of zenith
     in the principal plane (vSmartMOM convention).
   - `vaz = 180°` → Sun directly **behind** the satellite (same side
     of zenith). This is the `RAA = 0` convention used by MODTRAN and
     many other models.

2. **Added an azim180 convention-probe arm** of the EMIT/MODTRAN test:
   - Four new YAMLs in `test/test_parameters/`:
     `ParamsEMIT_MODTRANcomp_newLUT_azim180.yaml`,
     `..._alb02_azim180.yaml`, `..._IQU_azim180.yaml`,
     `..._IQU_alb02_azim180.yaml`. Each is identical to its parent
     except `vaz = 81.49773971061641 → -98.50226028938359` (parent − 180°).
   - Three parallel plotters in `test/benchmarks/`:
     `plot_MODTRAN_vSmartMOM_4EMIT_azim180.jl`,
     `plot_MODTRAN_vec_vSmartMOM_4EMIT_azim180.jl`,
     `plot_scalar_vs_vector_vSmartMOM_4EMIT_azim180.jl`. Read from
     `~/data/EMIT_MODTRANcomp/azim180/vSmartMOM_out/`, write to
     `~/data/EMIT_MODTRANcomp/azim180/plots/`, `_azim180` tagged.
   - The azim180 plotters have all per-spectral-slice and
     vSmartMOM-only / MODTRAN-only sections wrapped in
     `if false  # DISABLED for azim180 ... end` so only the
     full-spectrum MODTRAN-vs-vSmartMOM panels are produced (per
     scope-trim earlier in the day).

3. **Fixed an H₂O-scaling bug** in
   `test/benchmarks/emit_modtran_noRS_scenarios.jl` and
   `..._alb02.jl`. The inner loop did
   `model.τ_abs[ib] .= τ_abs_base[ib] .* h2o`, but `τ_abs` is the
   summed absorption optical depth over all species (H₂O, O₂, CO₂,
   NO₂, SO₂, CH₄, O₃, N₂O, CO), so the H₂O multiplier was
   contaminating CO₂, O₂, etc. Fix: moved
   `apply_scenario! + model_from_parameters` inside the H₂O loop, so
   each (gndalt, h2o) gets a fresh model build with `params.q .* h2o`
   (which only scales H₂O via the parser's auto-derived H₂O VMR).
   Aerosol τ_aer scaling stays in the AOT inner loop (linear and
   exact).

4. **Re-ran both suites** with the fixed drivers. Master script:
   `~/data/EMIT_MODTRANcomp/azim180/run_full_pipeline.sh`. Logs and
   timing in `~/data/EMIT_MODTRANcomp/azim180/logs/`.

5. **Side fix**: `~/.ssh/config` now has a `Host github.com` block
   with `ProxyCommand none` to bypass the system-wide
   `sss_ssh_knownhostsproxy` for github traffic. `git push/pull` over
   SSH works again.

# vSmartMOM vs MODTRAN — Comparison Summary for the EMIT Team
*Suniti Sanghavi · 2026-04-30 · vSmartMOM.jl on branch `sanghavi-unified`*

## TL;DR

**Yes, vSmartMOM can replace MODTRAN for EMIT applications**, on three grounds:

1. **Speed.** Cross-section LUTs (22 species, 280–3500 nm, Δν = 0.01 cm⁻¹) build in ~61 min once on a single A100; thereafter, a full radiative-transfer scenario at 0.1 cm⁻¹ output resolution over the 350–2510 nm window runs in tens of seconds on GPU and in minutes on CPU. Forward + analytic Jacobian (linearized RT) costs only modestly more than forward alone — no finite-difference perturbation runs needed.
2. **Flexibility.** Spectral grid, atmospheric profile, surface BRDF, polarization order, quadrature scheme, and HITRAN edition are all user-controlled via a single YAML config. The LUTs themselves can be rebuilt at any spectral resolution, p–T grid, or species list without touching the RT code.
3. **User-friendliness.** Full open-source Julia stack, single-file YAML driver, and the developer (S. Sanghavi) is local at JPL for direct support and customization. Active development for vSmartMOM is currently happening at JPL.

There **are** non-trivial differences between the two codes' outputs in the current EMIT comparison runs. Most are now traceable to *convention* (how MODTRAN packages its results relative to user inputs) and *spectral averaging* (how MODTRAN reports per-wavelength quantities), rather than physics gaps in vSmartMOM. The remaining open items are flagged below.

## 2. Speed & flexibility — concrete numbers

| Task | vSmartMOM | MODTRAN (typical, EMIT comparison) |
|---|---|---|
| One-time HITRAN LUT build (22 species, 280–3500 nm, Δν=0.01 cm⁻¹, 10 T × 15 p) | **~61 min on one A100, ~40 GB total storage** ([HANDOFF_LUT_REBUILD_2026-04-23 §G](HANDOFF_LUT_REBUILD_2026-04-23.md)) | N/A — bundled |
| Full EMIT regression scan: 60 scenarios (3 H2OSTR × 1 AOT × 6 GNDALT × 2 TSZ, 350–2510 nm @ Δν=0.1 cm⁻¹, 53 layers, alb=0) | **594 s (≈10 min) on one A100** with parallel-VZA scheme (both TSZs in one rt_run); serial-VZA baseline was 834 s — **1.40× speedup, byte-identical R/T at vza=12° and within Float32 roundoff (≤1e−6 rel) at vza=2°** | typically tens of minutes to hours, varies with binning |
| Single forward RT call (one H2OSTR × one AOT × one GNDALT, all VZAs) | **~12 s on A100** post-warmup (parallel-VZA scheme) | n/a |
| Forward + analytic Jacobians (per layer × per species × per surface parameter) | **~1.2–2× forward cost** (single linearized RT pass, no FD perturbations) | not natively supported; requires FD |
| Spectral resolution adjustment (e.g. 0.1 → 0.01 cm⁻¹) | Change one line in YAML; recompute LUT once. Run-time RT scales linearly with grid length. | Recompile / rebundle MODTRAN, or convolve outputs |
| Polarization order (scalar / vector / full Stokes) | Single YAML knob; no code changes | Vector capability limited (DISORT-driven); not a primary use case |
| Adding a new species | Build one HITRAN LUT (~3 min/species on A100) and add to YAML | Wait for next MODTRAN release |
| Multiple viewing geometries per scenario | **Free**: list `vza`/`vaz` as paired vectors in YAML; one rt_run produces all directions simultaneously (each requested VZA is added as a zero-weight quadrature stream, so the angular discretization is exact). Adding a 2nd VZA cost only ~50 % over a 1-VZA run, vs 100 % for sequential reruns. | Each VZA requires a separate run |

The headline message for EMIT: **the LUT is the only one-time cost**. After that, scenarios are inexpensive enough to do million-scenario LUT scans (the current EMIT regression already does ~36–360 scenarios per run for retrieval-LUT generation and could go higher).

### Flexibility highlights useful for EMIT

- **Spectral resolution is user-defined**, not bundled with the code. Same model can produce monochromatic line-by-line spectra (Δν = 0.01 cm⁻¹) for line-by-line fits, or coarse output for retrieval ingest, by changing one YAML field.
- **Surface models** include Lambertian (scalar / spectral), RPV, Ross-Li, Cox-Munk (with whitecaps + shadowing), and Canopy. New BRDFs are ~50 lines of Julia + a YAML mapping.
- **Profile** is fully user-supplied (any p, T, q, VMR), not constrained to a built-in catalog.
- **Polarization** is a compile-time choice between scalar (Stokes_I), 3-vector (IQU), and 4-vector (IQUV).
- **Inelastic (Raman) scattering** can be turned on/off per run; matters for high-resolution UV-Vis retrievals (Ring effect) but is an opt-in.
- **Linearized RT** returns analytic Jacobians w.r.t. τ_aer, τ_abs, surface parameters in a single forward pass — important for retrieval algorithms.

## 3. Output differences — what we currently see and why

The four-quadrant comparison driver ([`test/benchmarks/plot_MODTRAN_vSmartMOM_4EMIT.jl`](../test/benchmarks/plot_MODTRAN_vSmartMOM_4EMIT.jl)) produces per-scenario plots of the standard 6S decomposition fields (`ρ_atm`, `T↓_dir`, `T↓_dif`, `T↑_dir`, `T↑_dif`, `S_atm`) at five spectral slices (UV, VIS, NIR, SWIR1, SWIR2). The principal differences observed are below, in roughly decreasing order of magnitude/importance.

### 3.1 MODTRAN carries a baseline τ floor (~0.2) that does not scale with user inputs

**Strongest single discrepancy when looked at via the downward-leg apparent τ.** Best-evidence numbers from the H2OSTR sensitivity comparison (alb=0, AOT=1×10⁻⁴, GNDALT=0, TSZ=168, generated 2026-04-30):

| 2100–2200 nm window, *median* τ derived from `td_dir` (`−μ_s · ln td_dir`) | MODTRAN | vSmartMOM τ_total |
|---|---|---|
| H2OSTR = 0.05 (nominally near-dry) | **0.206** | 9.0×10⁻⁴ |
| H2OSTR = 2.05 (nominally wet)      | **0.213** | 1.8×10⁻² |
| Ratio (2.05 / 0.05)                | **1.03×** | ~20× |

MODTRAN's `td_dir`-derived τ moves only 3 % between dry and wet scenarios in the 2100–2200 nm continuum, while vSmartMOM moves by the expected H2O-dominated factor of 20×. **This is direct evidence that MODTRAN's `td_dir` carries a baseline (~0.2 of vertical optical depth) in this window that is independent of the user `H2OSTR` and `AOT550` knobs.**

**The same comparison from the *upward-leg* `tu_dir` looks dramatically different.** MODTRAN's 6S decomposition only debits aerosol *extinction* on the solar (downward) leg's `td_dir`; the up-leg `tu_dir` from the Lambertian source debits only aerosol *absorption* (forward-scattered photons cancel by symmetry under an isotropic source). So the ~0.2 baseline that contaminates the downward leg is largely absent from the upward leg, making `−μ_v · ln tu_dir` a much fairer total-τ proxy for code intercomparison:

| 2100–2200 nm window, MODTRAN τ derived from `tu_dir` (`−μ_v · ln tu_dir`) | H2OSTR = 0.05 | H2OSTR = 2.05 | Ratio |
|---|---|---|---|
| median                                            | 0.0054        | 0.0125        | **2.3×** |
| mean                                              | 0.0102        | 0.0444        | **4.4×** |

Now MODTRAN looks H₂O-sensitive — it moves by a factor of 2.3× (median) / 4.4× (mean) over the same column change, and the absolute magnitudes (~0.013 median, ~0.044 mean at H2OSTR=2.05) are within a factor of ~2 of vSmartMOM's τ_total in the same window. The residual difference is consistent with MODTRAN's still-active default H₂O continuum baseline (a few × 10⁻³) plus aerosol-absorption contribution — both small enough that the overall H₂O response is now visible.

**Recommendation.** When constructing intercomparison metrics from MODTRAN, prefer the upward-leg apparent τ (`−μ_v · ln tu_dir`) over the downward-leg version. The downward leg is convention-contaminated by ~0.2 of aerosol extinction baseline; the upward leg is not. vSmartMOM's `τ_total` should be compared to MODTRAN's upward-leg τ directly, or — even cleaner — both codes should report the H₂O-induced increment `Δτ = τ(H2OSTR_hi) − τ(H2OSTR_lo)`, which is convention-robust on either leg.

The most likely culprit for the downward-leg baseline is MODTRAN's default rural-23-km-visibility aerosol model, which we believe is being applied *additively* on top of the user's `AOT550` perturbation rather than replacing it. This is consistent with three other independent observations:

- A wavelength-independent ~0.20 offset between MODTRAN's `−μ_s · ln(td_dir)` and `−μ_v · ln(tu_dir)` across 380–3000 nm (see HANDOFF_LUT_REBUILD_2026-04-23 §I), which is the spectral signature of a flat-ish aerosol baseline appearing on the solar leg only under MODTRAN's 6S-style direct/diffuse decomposition.
- The O₂ A-band (759–763 nm) has identical apparent τ at H2OSTR=0.05 and 2.05 in MODTRAN (0.647 in both), as physics requires (O₂ column doesn't depend on H2OSTR) — this confirms MODTRAN's machinery is internally consistent.
- vSmartMOM's tiny SWIR2 τ at H2OSTR=0.05 is *correct* — at AOT=1×10⁻⁴, near-zero water column, and the surface albedo set to 0, the only remaining non-trivial absorber is MT_CKD self-continuum, which scales as vmr_h2o² and is therefore very small at H2OSTR=0.05.

**Implication for EMIT comparisons.** The previously-flagged "vSmartMOM is 3–7× too transparent in 2100–2200 nm" was an artifact of comparing the wrong MODTRAN field. The apples-to-apples options are (a) compare against MODTRAN's upward-leg `tu_dir` τ instead of `td_dir` τ; (b) recompute MODTRAN with `IHAZE=0` and any default continuum disabled; or (c) report H₂O-induced increments `Δτ = τ(H2OSTR_high) − τ(H2OSTR_low)` instead of raw τ.

### 3.2 The 6S "direct vs diffuse" convention: τ_dn ≠ τ_up in MODTRAN, τ_dn ≡ τ_up in vSmartMOM

For any monochromatic wavelength, Beer-Lambert says `−μ_s · ln(td_dir) = −μ_v · ln(tu_dir) = τ_total` should hold on both legs. **vSmartMOM satisfies this by construction** (both legs come from the same per-layer τ; `MODTRAN_vSmartMOM_4EMIT_tau_diff_*.pdf` shows the difference is identically zero). **MODTRAN does not.** Our observation:

| λ (nm) | MODTRAN τ_dn − τ_up |
|---|---|
| 380 | +0.20 |
| 550 | +0.20 |
| 760 (O₂ A-band) | **−0.21 (reversed)** |
| 1270 | +0.19 |
| 2200 | +0.21 |
| 3000 | +0.20 |

There are two distinct mechanisms:

- **The flat ~+0.20 offset across UV–VIS–NIR–SWIR** is MODTRAN's 6S aerosol convention: the solar-leg "direct" transmittance debits the full aerosol *extinction* (scattering + absorption), while the up-leg "direct" transmittance from the Lambertian surface debits only aerosol *absorption* (forward-scattered photons cancel by symmetry under an isotropic source). The aerosol baseline (§3.1) appears only on the down leg's `td_dir`, hence the offset.
- **The O₂-A-band reversal** is a curve-of-growth / spectral-averaging artifact: MODTRAN's per-bin output averages over multiple saturated O₂ lines, and saturated band-averaged transmittances grow super-Beer-Lambert with airmass. vSmartMOM samples at 0.1 cm⁻¹ (essentially monochromatic), so its bin-averaged values are still in the linear regime per line.

**The offset is constant at 0.20 across the full 240-scenario MODTRAN regression grid.** We computed the median per-scenario `τ_dn − τ_up` (excluding line-saturated wavelengths) for every available MODTRAN file. Result: every single one of 240 scenarios gives an offset that rounds to **0.20** at two decimals; the full spread across the grid is 0.2000–0.2007 (Δ = 0.0007).

| H2OSTR | TSZ (VZA) | n_scen (over AOT × GNDALT) | median offset | mean | min | max |
|---|---|---|---|---|---|---|
| 0.05 | 168° (12°) | 24 | 0.2005 | 0.2005 | 0.2004 | 0.2006 |
| 0.05 | 178° (2°)  | 24 | 0.2006 | 0.2006 | 0.2005 | 0.2007 |
| 0.55 | 168°       | 24 | 0.2003 | 0.2003 | 0.2001 | 0.2006 |
| 0.55 | 178°       | 24 | 0.2005 | 0.2005 | 0.2004 | 0.2007 |
| 1.05 | 168°       | 24 | 0.2003 | 0.2003 | 0.2000 | 0.2006 |
| 1.05 | 178°       | 24 | 0.2005 | 0.2005 | 0.2004 | 0.2007 |
| 1.55 | 168°       | 24 | 0.2003 | 0.2003 | 0.2000 | 0.2006 |
| 1.55 | 178°       | 24 | 0.2005 | 0.2005 | 0.2004 | 0.2007 |
| 2.05 | 168°       | 24 | 0.2003 | 0.2003 | 0.2000 | 0.2006 |
| 2.05 | 178°       | 24 | 0.2005 | 0.2005 | 0.2004 | 0.2007 |

The variations are sub-0.001 and physically interpretable but tiny: TSZ=178° (near-nadir, μ_v=0.999) is consistently ~0.0002 higher than TSZ=168° (μ_v=0.978), as expected from the geometry-driven `(μ_s·μ_v)/(μ_s−μ_v)` weighting of an aerosol extinction baseline; H2OSTR has zero effect; GNDALT and AOT each move the offset by ≤ 0.0002. **The user-knob `AOT550` does not modulate this offset.** That is direct evidence — independent of §3.1 — that MODTRAN is applying a fixed aerosol baseline (most plausibly the rural-23km default) regardless of the user's AOT550 directive.

**Implication for EMIT.** The standard 6S regression metric `T_tot = T_dir + T_dif` (which is convention-independent) is the right comparison field, not `T_dir` alone. We've been using `T_tot` for the alb=0 / alb=0.2 closure since 2026-04-24 ([HANDOFF_LUT_REBUILD_2026-04-23.md §B Decision 1](HANDOFF_LUT_REBUILD_2026-04-23.md)). Equivalently: the upward-leg apparent τ from `tu_dir` (which sees no offset) is a much fairer total-τ proxy for direct intercomparison than the downward-leg `td_dir`.

### 3.3 Spectral resolution and possible MODTRAN convolution

The single biggest uncertainty in the comparison is **how the MODTRAN .dat outputs were spectrally averaged or convolved.**

- **vSmartMOM** evaluates the LUT at exactly the user spec_bands (Δν = 0.1 cm⁻¹ in the current setup). At 2150 nm this corresponds to Δλ ≈ 0.046 nm; at 760 nm, Δλ ≈ 0.006 nm; at 350 nm, Δλ ≈ 0.0012 nm. This is essentially monochromatic — line cores are resolved, no instrument function applied.
- **MODTRAN**, depending on the run mode (BAND vs LINE-BY-LINE) and any post-processing applied before the `.dat` files were generated, may report:
  - true per-bin RT outputs (similar resolution to vSmartMOM in a given window), **or**
  - line-by-line outputs convolved with a Gaussian instrument function and resampled to a coarser grid, **or**
  - SLI-band model averages (1 cm⁻¹ or 5 cm⁻¹).

**The diagnostic signature**: in narrow saturated absorption bands, vSmartMOM's line cores reach larger τ than MODTRAN's; in continuum windows the two agree (apart from the baseline of §3.1). The 1.94 μm H₂O band is the cleanest case — vSmartMOM resolves individual line cores down to T_dir ≈ 0.005, while MODTRAN's binned T_dir bottoms out around 0.3–0.6 in the same band ([HANDOFF_LUT_REBUILD_2026-04-23 §H](HANDOFF_LUT_REBUILD_2026-04-23.md), 1930 nm row).

**Open question for the EMIT team**: was the MODTRAN output convolved (and at what FWHM), and against which slit function? **Without this information, we cannot fairly compare line-core depths between the two models.** The simplest path forward is to ask the MODTRAN run owner for the exact `tape5` and any `BNDWID` / `FWHM` / convolution settings.

If MODTRAN was run at full line-by-line resolution and binned to (say) 0.1-nm Gaussian, we can replicate the convolution on the vSmartMOM side trivially (Julia one-liner) before comparing. **This is the lowest-hanging-fruit reconciliation.**

### 3.4 CIA and water-vapor continuum — closed gaps

Earlier vSmartMOM versions had visible CIA and MT_CKD gaps:

- **1.27 μm (O₂ a¹Δ_g + collider)**: CIA contributes ~5 % slant attenuation in that window. vSmartMOM now ingests HITRAN-CIA 2024 (`O2-O2`, `O2-N2`, `N2-N2`) and reproduces the band depth.
- **1.9–2.7 μm (H₂O self + foreign continuum)**: vSmartMOM now uses MT_CKD 4.2 (`absco-ref_wv-mt-ckd.nc`). After this addition the SWIR2 gap shrunk modestly but did **not** close — most of the residual is the §3.1 baseline issue.

The cleanest demonstration that vSmartMOM's H₂O cross-sections themselves are correct is the H2OSTR=0.05 ↔ 2.05 sensitivity test (run 2026-04-30): vSmartMOM's 1.94 μm band deepens by a factor of ~20 between the two scenarios with all the right physical band morphology (see plots in `~/data/EMIT_MODTRANcomp/plots/h2o_sensitivity/`). MODTRAN does the same physically.

### 3.5 HITRAN edition mismatch — small effect, worth pinning

vSmartMOM's LUTs are built from **HITRAN 2024** (current default of `fetch_hitran`). MODTRAN's spectroscopy is bundled with the executable and is typically **HITRAN 2016 or 2020**. The differences for H₂O, O₂, CO₂ and CH₄ between adjacent HITRAN editions are at the ≤2 % band-integrated level, but can be sharper at individual line centers. **Action**: confirm the MODTRAN edition for the EMIT runs and rebuild a parallel vSmartMOM LUT against the matching edition if the EMIT team wants strictly version-matched science output. The rebuild script accepts an `edition` parameter, so this is a one-line change + 61 minutes of GPU time.

## 4. Open questions for the EMIT team

1. **MODTRAN tape5 archive.** Can the `.dat` files be paired with their generating tape5 inputs? In particular: `IHAZE`, `H2OSTR` semantics (full-column scale or perturbation?), `BNDWID`/`FWHM` convolution settings, HITRAN edition, and whether MT_CKD was active.
2. **MODTRAN's interpretation of `AOT550=1×10⁻⁴`.** Is this a *replacement* of the default rural aerosol model (so τ_aer ≈ 0 across the spectrum), or a *perturbation* on top of an internal default that stays at ~0.2? Confirming this would resolve §3.1.
3. **MODTRAN's interpretation of `H2OSTR=0.05`.** Does this scale the entire water column to 5 % of default, or does some clamped/baseline H₂O remain that ignores user input? The 3 % τ-window-median change between H2OSTR=0.05 and 2.05 strongly suggests the latter.
4. **Output convolution kernel.** If a Gaussian or boxcar was applied post-RT, what FWHM / bandwidth, and over what range? With this we can convolve vSmartMOM's outputs identically and produce a true apples-to-apples spectral comparison.
5. **EMIT science requirements on radiance accuracy.** What is the target retrieval precision (e.g. 0.1 % radiance, 1 % column water vapor)? This determines whether the existing residuals are blockers or noise-floor.

## 5. Recommended next steps

1. **Spectral convolution match.** Replicate MODTRAN's reported FWHM in vSmartMOM via a post-RT 1-D Gaussian convolution, regenerate the comparison plots — likely closes most of the line-core depth gap visible in current plots.
2. **Audit MODTRAN's tape5.** §4 items 1–3.
3. **Targeted sensitivity studies.** Re-run a single MODTRAN scenario with `IHAZE=0` and any continuum disabled to confirm or refute §3.1.
4. **Build a HITRAN-edition-matched LUT** if EMIT science requires version control.
5. **Lock the regression protocol** to compare H₂O-induced increments (Δτ between H2OSTR scenarios) and total-transmittance `T_tot = T_dir + T_dif` rather than raw `T_dir`, both of which are convention-robust.

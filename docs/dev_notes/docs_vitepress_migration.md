# DocumenterVitepress Migration Plan

Created: 2026-04-28
Status: **deferred** — do not start until [codex_next_task.md](codex_next_task.md) (warnonly retirement) is signed off.

Companion: [design_brief.md](design_brief.md) — the visual system this migration enables.

## Goal

Swap the docs renderer from Documenter's default HTML to
[DocumenterVitepress.jl](https://github.com/LuxDL/DocumenterVitepress.jl),
matching the visual quality of Oceananigans, Lux, and the SciML stack. Add a
proper landing page with a hero block, three-feature grid, and a coherent icon
set.

The content (markdown + Documenter directives) does not change. Only the
renderer, the landing page presentation, and the build pipeline change.

## Why

- Modern sidebar nav with collapsible groups — much better fit for the
  task-based / module-reference split that already exists.
- Local search out of the box (Algolia DocSearch as upgrade path).
- Dark mode, mobile-responsive, copy-button on code blocks.
- Vitepress hero block enables a real first impression instead of an H1 + paragraph.
- KaTeX math renders faster than MathJax with the same source markup.
- Closer visual register to other modern Julia scientific packages, which
  matters for credibility at registration time.

## Costs / risks

- Adds Node.js to the docs build (CI step + local dev requirement).
- DocumenterVitepress is at a young 0.x/1.x, version-pin needed.
- Asset paths and gh-pages output layout differ slightly — link consumers may
  break. Audit external inbound links before cutover.
- If the warnonly retirement isn't done first, the renderer change will mask
  whether new warnings come from content or from the new toolchain.

## Files touched (cutover commit)

- `docs/Project.toml` — add `DocumenterVitepress` with version pin
- `docs/make.jl` — swap `Documenter.HTML(...)` for `DocumenterVitepress.MarkdownVitepress(...)`; adjust deploy block
- `docs/src/.vitepress/config.mts` — generated by DocumenterVitepress on first build, then customized for sidebar grouping
- `.github/workflows/Documentation.yml` — add Node.js setup step before Julia build
- `docs/src/index.md` — frontmatter hero block (content stays as-is below the fold)

That's the **renderer cutover**. The hero/icon/feature work is a separate follow-up commit per [design_brief.md](design_brief.md).

## Step-by-step

### Step 1 — Add the dependency

```julia
# docs/Project.toml
[deps]
DocumenterVitepress = "4710194d-e776-4893-9690-8d956a29c365"
# ...

[compat]
DocumenterVitepress = "0.1, 0.2"  # pin a known-good range; check latest at PR time
```

### Step 2 — Swap the renderer in `make.jl`

Replace the current `Documenter.HTML(...)` block with:

```julia
using DocumenterVitepress

format = MarkdownVitepress(
    repo = "github.com/RemoteSensingTools/vSmartMOM.jl",
    devbranch = "main",
    devurl = "dev",
    deploy_url = "remotesensingtools.github.io/vSmartMOM.jl",
)
```

The `pages = Any[...]` block stays. DocumenterVitepress reads it and generates
the corresponding Vitepress sidebar.

The `deploydocs(...)` call stays unchanged.

### Step 3 — CI workflow

Add a Node.js setup step before the Julia build in `.github/workflows/Documentation.yml`:

```yaml
- uses: actions/setup-node@v4
  with:
    node-version: '20'
- name: Install Vitepress
  run: |
    cd docs
    npm install
```

DocumenterVitepress generates a `package.json` on first build; commit that.

### Step 4 — First local build

```bash
julia --project=docs --startup-file=no docs/make.jl
```

This generates `docs/src/.vitepress/config.mts` and a `package.json` in
`docs/`. Commit both. The first build will probably need theme tweaks — those
go in follow-up commits, not this one.

### Step 5 — Sidebar polish

DocumenterVitepress derives the sidebar from `pages = Any[...]`. The current
two-layer split (task-based on top, module reference below) maps cleanly onto
Vitepress sidebar sections. If the auto-generated grouping doesn't match,
edit `docs/src/.vitepress/config.mts`'s `sidebar` array directly.

Target structure:

```
Get Started
  Quick Start
  Configure a Scene
Use
  Compute Jacobians
  Run on GPU
Extend
  Add a Surface BRDF
  Add a Raman Mode
Reference
  Architecture & Design
  Core RT Theory
  API Reference
  Release Notes
Modules
  CoreRT
  Absorption
  Scattering
  Inelastic
  Aerosols
  SolarModel
  Surfaces
  IO Reference
Tutorials
  ...
```

This is a tightening of the current nav, not new content.

## Validation

- `julia --project=docs --startup-file=no docs/make.jl` exits 0
- `docs/build/index.html` opens locally and renders a Vitepress hero
- `docs/test_examples.jl` still passes
- All sidebar entries resolve to a real page
- Search box appears top-right and finds known terms
- Dark mode toggle works
- Mobile layout (Chrome devtools, 375px width) is usable

## Rollback

If the cutover regresses something critical, the rollback is one commit:
revert the cutover commit, drop `DocumenterVitepress` from `Project.toml`,
restore the old `Documenter.HTML(...)` block. Content is unchanged so nothing
else needs touching.

## Out of scope for the cutover commit

These come after, in separate small commits:

- Hero frontmatter on `index.md` (per [design_brief.md](design_brief.md))
- Icon SVGs in `docs/src/assets/icons/`
- Custom theme colors / fonts

- Custom 404 page

Keep the cutover surgical so a regression is bisectable.
